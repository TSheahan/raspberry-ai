# Platform notes
# --------------
# pyalsaaudio
#   Direct ALSA bindings (snd_pcm_writei). Used for TTS audio output on Pi.
#   Replaces PyAudio — PortAudio's callback thread causes tearing on bcm2835.
#   Linux-only; Windows fallback uses PyAudio. Build requires libasound2-dev.
#   See profiling-pi/venv.md for install/verify steps.
#
# PyAudio (0.2.14)
#   C extension wrapping PortAudio. Retained for Windows dev fallback only.
#   On Pi: superseded by pyalsaaudio for audio output (tearing fix, 2026-04-05).
#   On Windows: pre-built wheel available on PyPI for Python 3.13 — installs
#   cleanly with no PortAudio SDK required (verified 2026-04-02).
#
# torch (2.11.0)
#   Pi freeze records 2.11.0+cpu — a local version label from PyTorch's CPU
#   wheel index, not a standard PyPI qualifier. Heavy (~2 GB); not required
#   for IDE source navigation of the harness itself.
#   To install: pip install torch==2.11.0 --index-url https://download.pytorch.org/whl/cpu
#
# audioop-lts (0.2.2)
#   Backport of the audioop C module removed from stdlib in Python 3.13.
#   Required by PyAudio on Python 3.13+.
