# Stage 1 — v01a: Gate OWW During Cognitive Loop

**Base file:** `voice_pipeline_step7_v01.py`
**Output file:** `voice_pipeline_step7_v01a.py` (copy v01, apply changes below)
**Sequence:** 1 of 4

---

## Problem This Solves

In v01, `OpenWakeWordProcessor.process_frame` calls `self.model.predict()` on every audio chunk,
including during the 7–12s cognitive loop (STT + Claude subprocess). OWW's ONNX inference
(20–40ms per call on Pi 4) competes with the Claude subprocess for CPU. Frames accumulate in
unbounded queues. Ctrl+C after a cognitive loop → queue flush → USB audio cascade → Pi reboot.

This stage gates OWW only. VAD (Silero) is unchanged — it still runs on every frame.

## Change: OpenWakeWordProcessor.process_frame (v01.py lines 71–94)

**v01 original (lines 71–94):**

```python
async def process_frame(self, frame: Frame, direction):
    await super().process_frame(frame, direction)
    if isinstance(frame, AudioRawFrame):
        audio_chunk = np.frombuffer(frame.audio, dtype=np.int16)
        self._chunks.append(audio_chunk)
        buffer = np.concatenate(self._chunks)
        chunk_size = 1280
        consumed = 0
        while len(buffer) - consumed >= chunk_size:
            chunk = buffer[consumed:consumed + chunk_size]
            consumed += chunk_size
            predictions = self.model.predict(chunk.astype(np.float32))
            current_time = time.time()
            for wakeword, score in predictions.items():
                if (wakeword == "hey_jarvis"
                        and score > 0.5
                        and (current_time - self.last_detection_time) > self.DEBOUNCE_SECONDS):
                    print(f"\nWAKE DETECTED -- '{wakeword}'  |  score: {score:.3f}")
                    self.last_detection_time = current_time
                    self.capturer.start_capture()
        remainder = buffer[consumed:]
        self._chunks = [remainder] if len(remainder) > 0 else []
    await self.push_frame(frame, direction)
```

**v01a replacement:**

```python
async def process_frame(self, frame: Frame, direction):
    await super().process_frame(frame, direction)
    if isinstance(frame, AudioRawFrame):
        # Skip predict while capturing or processing — CPU contention with
        # Silero VAD and the Claude subprocess causes queue accumulation.
        if self.capturer.processing or self.capturer.capturing:
            self._chunks = []
            await self.push_frame(frame, direction)
            return
        audio_chunk = np.frombuffer(frame.audio, dtype=np.int16)
        self._chunks.append(audio_chunk)
        buffer = np.concatenate(self._chunks)
        chunk_size = 1280
        consumed = 0
        while len(buffer) - consumed >= chunk_size:
            chunk = buffer[consumed:consumed + chunk_size]
            consumed += chunk_size
            predictions = self.model.predict(chunk.astype(np.float32))
            current_time = time.time()
            for wakeword, score in predictions.items():
                if (wakeword == "hey_jarvis"
                        and score > 0.5
                        and (current_time - self.last_detection_time) > self.DEBOUNCE_SECONDS):
                    print(f"\nWAKE DETECTED -- '{wakeword}'  |  score: {score:.3f}")
                    self.last_detection_time = current_time
                    self.capturer.start_capture()
        remainder = buffer[consumed:]
        self._chunks = [remainder] if len(remainder) > 0 else []
    await self.push_frame(frame, direction)
```

**What changed:** 5 lines added after `if isinstance(frame, AudioRawFrame):` — gate check,
chunk clear, push, return. Everything else is identical to v01.

## Expected Behavior

1. "Listening for wake word..." prints on startup
2. Say "hey Jarvis" → `WAKE DETECTED` prints (OWW running normally in LISTENING phase)
3. OWW goes quiet during capture + cognitive loop (no predict calls)
4. VAD fires when speech ends → `VADUserStoppedSpeakingFrame` → cognitive loop starts
5. Claude responds, latency printed, returns to listening
6. Ctrl+C: cleaner than v01, but Silero VAD still runs on all frames during capturing —
   may still reboot if loop is long enough

---

## What to Watch For

- **OWW gating confirmed:** no `WAKE DETECTED` during cognitive loop unless you actually
  say the wake word (shouldn't happen mid-cognitive loop)
- **VAD still fires:** `VADUserStoppedSpeakingFrame` must still arrive at `UtteranceCapturer`
  — if VAD never triggers, the cognitive loop never starts. This would indicate a problem with
  Silero, not with this stage's change.
- **Reboot risk:** lower than v01 (OWW gated) but not eliminated (Silero still unbounded).
  If reboot occurs, it confirms Silero is the remaining CPU pressure — proceed to v01b.
