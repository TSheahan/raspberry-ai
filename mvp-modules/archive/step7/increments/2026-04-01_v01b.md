## Problem This Solves

In v01a, `VADProcessor` (Pipecat built-in) runs Silero ONNX inference on **every** audio frame
throughout all pipeline phases. During the cognitive loop (STT + Claude, 7–12s), Silero
(15–25ms/frame) runs alongside the Claude subprocess — two concurrent CPU loads, queues grow,
potential reboot on Ctrl+C.

This stage replaces `VADProcessor` with a custom `GatedVADProcessor` that wraps the same
Silero analyzer but only feeds frames to it during `capturer.capturing`. Outside that window,
audio frames are forwarded downstream without touching ONNX.

This is the **first introduction** of `GatedVADProcessor` — structural only, no prints.
The `_vad_frame_count` attribute is initialized here (fixing the latent v02 bug before it
can bite).

## New Class: GatedVADProcessor

Insert before `OpenWakeWordProcessor`. This replaces the built-in `VADProcessor`.

```python
class GatedVADProcessor(FrameProcessor):
    """VAD that only runs Silero inference during CAPTURING phase.

    Audio frames are gated — Silero only sees them when capturer.capturing is True.
    All non-audio frames always reach the VAD controller (lifecycle events).
    """

    def __init__(self, *, vad_analyzer, capturer, **kwargs):
        super().__init__(**kwargs)
        self._capturer = capturer
        self._vad_analyzer = vad_analyzer
        self._vad_controller = VADController(vad_analyzer)
        self._vad_frame_count = 0   # initialized here — v02 omits this, causing AttributeError

        @self._vad_controller.event_handler("on_push_frame")
        async def on_push_frame(_controller, frame, direction):
            await self.push_frame(frame, direction)

        @self._vad_controller.event_handler("on_broadcast_frame")
        async def on_broadcast_frame(_controller, frame_cls, **kw):
            await self.broadcast_frame(frame_cls, **kw)

    async def process_frame(self, frame: Frame, direction: FrameDirection):
        await super().process_frame(frame, direction)
        await self.push_frame(frame, direction)   # always forward downstream

        if isinstance(frame, (AudioRawFrame, InputAudioRawFrame)):
            # Gate: only run Silero during capture window
            if self._capturer.capturing:
                self._vad_frame_count += 1
                await self._vad_controller.process_frame(frame)
        else:
            # Non-audio frames always reach controller (StartFrame, EndFrame, etc.)
            await self._vad_controller.process_frame(frame)
```

## What to Watch For

- **VAD fires during capture:** The cognitive loop only starts if `VADUserStoppedSpeakingFrame`
  arrives at `UtteranceCapturer`. If it never arrives, check that `_vad_controller` is seeing
  frames. The `_vad_frame_count` counter increments but isn't printed yet — add a temporary
  `print(self._vad_frame_count)` inside the gate block if needed.
- **No WAKE DETECTED:** OWW gating from v01a is still present — if `capturer.capturing` is
  somehow stuck True, OWW won't run. Check `capturer.capturing` state.
- **Reboot risk:** Should be substantially reduced — both ONNX workloads now gated to
  non-overlapping phases. If reboot still occurs, check whether `_cognitive_loop` task is
  completing and clearing `processing = False`.
