## Why This Stage Exists

When VAD breaks silently (wrong frame gating, AttributeError in handler, Silero state issue),
the pipeline appears to work — wake word fires, audio is captured — but the cognitive loop
never starts because `VADUserStoppedSpeakingFrame` never arrives. The only diagnostic signal
is that `[VAD] speech_started` and `[VAD] speech_stopped` stop printing. Without these prints,
the failure is invisible.

These prints are the minimum observability needed to distinguish:
- "VAD is running and detecting speech" (both print)
- "VAD is running but not seeing frames" (neither prints)
- "VAD fires speech_started but not speech_stopped" (timeout/threshold issue)

## Change: Add Event Handlers to GatedVADProcessor.__init__

In v01b, `GatedVADProcessor.__init__` has `on_push_frame` and `on_broadcast_frame` handlers
but no speech event handlers. Add `on_speech_started` and `on_speech_stopped` between the
`VADController` construction and the existing push/broadcast handlers.

**Add after `self._vad_frame_count = 0`:**

```python
        @self._vad_controller.event_handler("on_speech_started")
        async def on_speech_started(_controller):
            print(f"  [VAD] speech_started ({self._vad_frame_count} frames to Silero so far)")

        @self._vad_controller.event_handler("on_speech_stopped")
        async def on_speech_stopped(_controller):
            print(f"  [VAD] speech_stopped ({self._vad_frame_count} frames to Silero total)")
```

`self._vad_frame_count` is already initialized in v01b — this is the fix that v02.py omits.

## Full GatedVADProcessor.__init__ After Change

```python
    def __init__(self, *, vad_analyzer, capturer, **kwargs):
        super().__init__(**kwargs)
        self._capturer = capturer
        self._vad_analyzer = vad_analyzer
        self._vad_controller = VADController(vad_analyzer)
        self._vad_frame_count = 0

        @self._vad_controller.event_handler("on_speech_started")
        async def on_speech_started(_controller):
            print(f"  [VAD] speech_started ({self._vad_frame_count} frames to Silero so far)")

        @self._vad_controller.event_handler("on_speech_stopped")
        async def on_speech_stopped(_controller):
            print(f"  [VAD] speech_stopped ({self._vad_frame_count} frames to Silero total)")

        @self._vad_controller.event_handler("on_push_frame")
        async def on_push_frame(_controller, frame, direction):
            await self.push_frame(frame, direction)

        @self._vad_controller.event_handler("on_broadcast_frame")
        async def on_broadcast_frame(_controller, frame_cls, **kw):
            await self.broadcast_frame(frame_cls, **kw)
```

No other changes to `GatedVADProcessor`.

## What to Watch For

- **`[VAD] speech_started` prints but `speech_stopped` never prints:** Silero is detecting
  speech but not silence. Increase `stop_secs` (try 1.2 or 1.8). Or user stopped speaking
  but threshold not met.
- **Neither speech_started nor speech_stopped print after wake:** `_vad_controller` is not
  receiving frames during capturing. Check the gate condition in `process_frame`.
- **`speech_stopped` prints but cognitive loop never starts:** `VADUserStoppedSpeakingFrame`
  is not reaching `UtteranceCapturer`. The `on_speech_stopped` handler fires but the frame
  emission fails. Check `on_push_frame` and `on_broadcast_frame` handler wiring.
- **AttributeError in handler:** would appear in console. Means `_vad_frame_count` is missing —
  should not happen as of v01b, but check that the attribute init line is present.
- **Frame count is 0 in both prints:** speech events fired but no frames were counted.
  Means `capturing` was False when frames arrived — gate timing issue.
