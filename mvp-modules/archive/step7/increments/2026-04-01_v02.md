## Known Differences (v01c vs v02.py)

These are expected and intentional — v01c is correct, v02.py has bugs:

### 1. `_vad_frame_count` initialization

**v01c (correct):**
```python
self._vad_frame_count = 0   # in __init__
```

**v02.py (bug — omits this line):**
`_vad_frame_count` is referenced in `on_speech_started`/`on_speech_stopped` but never
initialized. First speech detection → `AttributeError` → VAD silently dead.

**Resolution:** Keep v01c. Do not add the missing init to v02.py — v01c IS the fixed v02.

### 2. Audio-only gating in process_frame

**v01c (correct — from v07 pattern):**
```python
if isinstance(frame, (AudioRawFrame, InputAudioRawFrame)):
    if self._capturer.capturing:
        await self._vad_controller.process_frame(frame)
else:
    await self._vad_controller.process_frame(frame)
```

**v02.py (partial — misses non-audio forwarding):**
```python
if isinstance(frame, StartFrame):
    await self._vad_controller.process_frame(frame)
elif self._capturer.capturing:
    await self._vad_controller.process_frame(frame)
```

v02 only forwards `StartFrame` and audio-when-capturing. All other non-audio frames
(EndFrame, CancelFrame, SystemFrame) never reach the controller.

**Resolution:** Keep v01c. The v07 pattern (audio-only gate + else-all) is more robust.

### 3. print format in speech event handlers

**v01c:**
```python
print(f"  [VAD] speech_started ({self._vad_frame_count} frames to Silero so far)")
print(f"  [VAD] speech_stopped ({self._vad_frame_count} frames to Silero total)")
```

**v02.py:**
```python
print(f"  [VAD] speech_started (after {self._vad_frame_count} frames to Silero)")
print(f"  [VAD] speech_stopped (after {self._vad_frame_count} frames to Silero)")
```

**Resolution:** Inconsequential. Match v02.py wording if preferred, or keep v01c phrasing.
